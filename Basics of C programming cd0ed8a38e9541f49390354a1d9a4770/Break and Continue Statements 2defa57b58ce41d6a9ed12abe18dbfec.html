Break and Continue Statements

Introduction to Break Statement

The break statement in C programming is used to immediately terminate the execution of a loop (for, while, do-while) or switch-case statement. It transfers control to the statement immediately following the terminated loop or switch-case block.

Syntax of Break Statement:


break;

Example of Break Statement:


#include <stdio.h>int main() {
    int i;

    // Print numbers from 1 to 10, but stop at 5
    for (i = 1; i <= 10; i++) {
        if (i == 5) {
            break; // Terminate the loop when i equals 5
        }
        printf("%d ", i);
    }
    printf("\n");

    return 0;
}

Explanation: In this example, the for loop iterates from 1 to 10. When i equals 5, the break statement is encountered, causing the loop to terminate immediately.

Usage of Break Statement:

  • Exit a loop prematurely when a certain condition is met.
  • Provide an escape mechanism from an infinite loop or unwanted iterations.

Introduction to Continue Statement

The continue statement in C programming is used to skip the remaining code in the current iteration of a loop and move to the next iteration. It is often used within loops to skip processing for certain conditions without terminating the loop altogether.

Syntax of Continue Statement:


continue;

Example of Continue Statement:


#include <stdio.h>int main() {
    int i;

    // Print only even numbers from 1 to 10
    for (i = 1; i <= 10; i++) {
        if (i % 2 != 0) {
            continue; // Skip odd numbers
        }
        printf("%d ", i);
    }
    printf("\n");

    return 0;
}

Explanation: In this example, the for loop iterates from 1 to 10. The continue statement skips odd numbers (i % 2 != 0) and proceeds to the next iteration to print only even numbers.

Usage of Continue Statement:

  • Skip certain iterations of a loop based on specific conditions.
  • Improve efficiency by avoiding unnecessary computations or processing.

Comparison of Break and Continue Statements:

  • Break: Terminates the loop immediately and transfers control to the statement following the loop.
  • Continue: Skips the remaining code in the current iteration and moves to the next iteration of the loop.

Conclusion
